Part B — Flexbox Two-Column Layout

B1) Identify the Flexbox parent

Question: What element do you think the flexbox parent is? Why?

💡 Hint: The flexbox parent is the container for the flex items. In this case, our flex items are the sidebar and the content.

Once you've identified the flexbox parent, add CSS to make it a flex container. This sets up your two-column layout.

Hints:

  • The flexbox parent: It's the .container class (on your <main> element). This element directly contains both the sidebar and content area.
  • Making it a flex container: Add display: flex; to the .container selector.
  • Add spacing: Use the gap property to create space between the sidebar and content (try gap: 1.5rem;).
  • Add padding: Give the container some breathing room with padding: 1rem;.

B2) Size the two columns

You want the sidebar to stay a fixed 300px and never get squished, and the content to take all the leftover space. That's just three lines of Flexbox:

.sidebar {
  width: 300px;
  flex-shrink: 0;
}

.content {
  flex: 1;
}

In plain English:

  • width: 300px; — the sidebar is 300px wide (a property you already know).
  • flex-shrink: 0; — never let it get squished below 300px.
  • flex: 1; on the content — take whatever space is left over.

That's the whole layout. Add a little styling to the sidebar so you can see it—reveal the hints if you want specifics.

Hints:

  • Sidebar styling: Add padding: 1rem;, a light background color like #f5f5f5, and border-radius: 8px; for rounded corners.
  • Content width: Optionally add max-width: 800px; to the content so long lines of text stay readable.
  • That's all the Flexbox you need here: display: flex on the parent (Part B1), flex-shrink: 0 on the fixed column, and flex: 1 on the flexible one.

B3) Going deeper (optional): what flex: 1 really means

You don't need this to finish the project—your layout already works. Expand it only if you're curious why those lines do what they do.

The longhand behind the shorthand

flex is shorthand for three properties bundled together. Writing flex: 1; is the same as writing:

flex-grow: 1;    /* expand to fill extra space */
flex-shrink: 1;  /* allowed to shrink if needed */
flex-basis: 0;   /* starting size before grow/shrink */
  • flex-grow — how greedily an item takes leftover space. 0 = never grow; 1 = take a share.
  • flex-shrink — whether an item may get smaller when space is tight. 0 = never shrink (that's exactly why our sidebar keeps its 300px).
  • flex-basis — the item's size before any growing or shrinking happens.
🎯 Our layout, fully spelled out: the sidebar is flex: 0 0 300px (don't grow, don't shrink, stay 300px) and the content is flex: 1 1 0 (grow, may shrink, start from nothing and fill the rest). The short versions you actually wrote—width: 300px + flex-shrink: 0, and flex: 1—are easier to read and do the same job. Reach for the long form only when you need fine control.

Part C — Sticky Sidebar (No JavaScript)

C1) Make the sidebar sticky

Now let's make the sidebar stay pinned while you scroll the page. This is pure CSS—no JavaScript needed!

⚠️ Common gotcha: Sticky positioning only works when the element's ancestor allows it (no overflow: hidden on parent elements). Keep .container simple!

Part D — Anchor Links + Scroll Polish

D1) See how the anchor links work

Your starter already wires the navigation for you—you don't build it. In the HTML panel, find one matching pair:

  1. A link in .sidebar with an href like href="#section-1"
  2. The <section> with the matching id="section-1"
  3. Now click that nav link in the preview—does the page jump to that section?
💡 How anchor links work: When you click a link with href="#section-1", the browser looks for an element with id="section-1" and scrolls to it. The # symbol tells the browser to look for an id on the current page rather than navigating to a new page. This is exactly why Part A warned you not to rename the section ids—renaming them would break this navigation.

D2) Smooth scroll + heading alignment

Right now, clicking anchor links causes an instant jump. Let's add smooth scrolling and prevent headings from being cut off at the top.

📚 Research these CSS properties:

  1. Smooth scrolling: Read about scroll-behavior on MDN
    • What element does this property go on? (html or your sections?)
    • What value creates smooth scrolling?
  2. Scroll margin: Read about scroll-margin-top on MDN
    • Why would you need space at the top when jumping to a section?
    • What's a good value to prevent headings from being glued to the edge?

Your task: Add CSS for smooth scrolling and proper spacing when jumping to sections. Also, add some visual styling to your sections (padding, background, borders, etc.).

Hints:

  • Smooth scrolling: Add scroll-behavior: smooth; to the html selector for smooth anchor navigation.
  • Style your sections: Target your section elements directly with padding: 2rem;, margin-bottom: 2rem;, background: white;, border: 1px solid #e0e0e0;, and border-radius: 8px;.
  • Prevent header cutoff: Add scroll-margin-top: 1rem; to your sections so headings aren't glued to the top edge when jumping to anchors.
  • Prepare for Part E: Add position: relative; to your sections—this will be needed for positioning decorative icons later.
📚 Resources:

Part E — Decorative Icons (Per Section)

E1) Download icons from The Noun Project

Let's add visual interest to each section with thematic icons related to flexbox and web design.

📥 Download icons:

  1. Go to The Noun Project
  2. Search for flexbox-related terms like:
    • "layout" or "grid"
    • "container" or "box"
    • "align" or "arrange"
    • "responsive" or "adapt"
    • "web design" or "code"
  3. Download 5 icons (one for each section) as PNG or SVG files
  4. Choose icons that complement your content themes
  5. Note: Free icons are available! Look for the "Free" filter or create a free account
💡 Icon tips:
  • Download in a consistent style (all line art, all filled, etc.)
  • Choose a reasonable size (128px or 256px works well)
  • SVG files are preferred—they scale perfectly and have smaller file sizes
  • Save them with descriptive names like icon-flexbox.svg, icon-container.svg, etc.

E2) Add icons to your HTML

Now let's add the icons to each section's deco div.

For CodePen users (using Snipboard):

  1. Go to Snipboard.io
  2. Click "Upload Files" and select your 5 icons
  3. Once uploaded, click each image and copy its direct URL
  4. In your HTML, update each deco div to include an <img> tag: 
    <div class="deco" id="deco-1">
  <img src="SNIPBOARD_URL_HERE" alt="">
</div>
  5. Repeat for all 5 sections with their respective icon URLs
⚠️ Important for Final Project: When you create your final project submission (not in CodePen), you'll need to:
  • Create an images folder in your project directory
  • Save all your icon files in that folder
  • Update your image paths to images/icon-name.svg
  • This ensures your images work when you submit your files or host your site

For local files (working outside CodePen):

  1. Create an images folder in your project directory
  2. Move your downloaded icons into that folder
  3. Reference them in your HTML:

  <img src="images/icon-flexbox.svg" alt="">

📝 Accessibility Note: We use alt="" (empty alt text) because these icons are purely decorative and don't convey essential information. Screen readers will skip them.

E3) Style the decorative icons

Style the icons so they add visual interest without interfering with content.

📚 Research CSS positioning:

  • Learn about pointer-events: none to prevent icons from blocking clicks

Your task: Style the .deco class and the img inside it:

  • Position decorations absolutely so they don't affect layout
  • Make them semi-transparent with opacity
  • Prevent them from blocking clicks with pointer-events: none
  • Size the images appropriately (try 150-200px)

Hints:

  • Position the deco container: Add position: absolute; to .deco so it doesn't affect layout flow
  • Make it non-blocking: Add pointer-events: none; so users can still click links beneath the icons
  • Style the images: Target .deco img and set:
    • width: 150px; (or your preferred size)

E4) Position each icon uniquely

Style each decoration differently using their unique ids. Place icons in different positions to create visual variety.

Your task: Use id selectors to position each icon differently:

  • Continue for remaining sections with your own creative positions!

Hints:

  • Target by id: Use #deco-1, #deco-2, etc. to style each decoration uniquely

Part F — Finishing Touches (Optional)

Optional Enhancements

Polish your project with these accessibility and usability improvements. Note: While optional, completing Part F demonstrates attention to accessibility and can earn you extra consideration in grading.

  • Add a focus style for sidebar links (:focus-visible)
  • Style the active link based on scroll (stretch goal with IntersectionObserver)
  • Add a "Back to top" link at the end of each section

Hints:

  • Focus styles: Target your nav links with :focus-visible pseudo-class. Add a visible outline (try 2px solid #3b82f6), outline-offset: 2px;, and border-radius: 4px; for polish.
  • Back to top link (HTML): Add <a href="#" class="back-to-top">↑ Back to top</a> at the end of each section.
  • Back to top link (CSS): Style .back-to-top with display: inline-block;, margin-top: 2rem;, padding: 0.5rem 1rem;, a background color, border-radius: 4px;, and remove underline with text-decoration: none;. Add a :hover state with a darker background.
  • Stretch goal - Active link: Use JavaScript's IntersectionObserver API to detect which section is currently visible and add an "active" class to the corresponding nav link.
