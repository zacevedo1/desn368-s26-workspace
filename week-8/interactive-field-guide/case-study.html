Design Decisions

I think most of the design decisions I made was based off of the word mark and the main character Maomao, you know when you watch a show and it just gives off this color or shade all through out the show? Kind of like that, but i also did try matching the colors to reflect the imperial court setting. I chose a deep maroon and forest green palette, rich, serious, and layered with meaning. The cream and gold accent tones were pulled directly from traditional East Asian manuscript aesthetics, which felt appropriate for a story set in a fictionalized historical empire. For the fonts I wanted something that was not too simple but still gave off historic vibes and so I went with Bona Nova SC as the primary and Laila as the body text.

I selected Bona Nova SC as the primary typeface because its small-caps character gave the headings a formal, archival quality without feeling stiff. Every section heading uses font-variant: small-caps, which ties the typography back to that historical reference without needing ornate decorative fonts. The sidebar navigation was designed to feel like a bookmark system in a physical guide, narrow, dark, and always present.

For the hero section, i wanted to create a sense of depth and flow for the banner image, and make it feel like the image was flowing into the design, so i edited the original image to have a gradient fade from the bottom up, doing this also allowed me to incorporate the word mark into the banner image itself, which was a fun design choice to make, and without it, the word mark would have been hard to see. The result gives the top of the page a more cinematic, editorial quality that sets the tone before the reader scrolls into the content.

Technical Challenges

The part that i had the most technical trouble with was the side bar and the hamburger menu. Since ive never done it before i had to do a lot of researching not only that but assistance by two different AI modeles, Gemini and Claude. Eventually i was able to figure it out but even now i feel like it still would not know how to do it again.

Another challenge was the active navigation state. I wanted the sidebar link to highlight green when the user was in the corresponding section. Active states are another area where i struggled on, especially figuring out the CSS so i did need to have the models help me with that as well. I ended up using the :target pseudo-class to style the active link based on the URL hash, which was a simple solution but has limitations since it doesn't track actual scroll position, why you ask? Because i would need javascript, and that unfortunately i have not learned how to use yet.

The character card grid also gave me trouble early on. Only two images were rendering visibly at first, caused by a combination of a global img { width: 30% } rule being removed and min-width: auto on flex children causing overflow clipping. The fix was adding min-width: 0 to the fig element and switching the image to width: 100%; max-width: 130px so it scaled within its grid cell correctly.

Responsive Strategy

I used two breakpoints: a mobile breakpoint at max-width: 640px and a large desktop breakpoint at min-width: 1440px. The mobile breakpoint handles the most significant layout shift, the sticky sidebar becomes a hidden off-canvas panel, the 4-column character grid collapses to 2 columns, the media images stack vertically, and the author section shifts from a horizontal to a vertical layout.

On mobile, I wanted users to still have access to navigation without the sidebar taking up the whole screen. The hamburger menu handles this with a fixed icon at the top-left. The title bar (header h1) is also fixed on mobile so the page title and brand stay visible while scrolling through long content sections. I kept the banner image and wordmark in the normal document flow so they appear at the top of the page when the user first lands.

The desktop breakpoint at 1440px widens the sidebar from 210px to 300px and increases the padding and heading font size for larger displays. This ensures the layout doesn't feel cramped or stretched at very large screen widths, while keeping a consistent visual weight across the content area.

Learning Moments

This project taught me a lot about the difference between position: sticky, position: fixed, and position: absolute in practice. This project forced me to think about how they interact, for example, why position: sticky stops working when a parent has overflow: hidden, or why a child set to position: absolute positions itself relative to the nearest positioned ancestor rather than the page. However keep in mind if you where to ask me what these are a day later i would have absolutly no idea.

Flexbox was the biggest challenge for me, because its one of those concepts that seems simple until you try to implement it, and when you think you have it figured out, you realize you don't. I did learn however that flex children have min-width: auto by default, which can cause them to overflow their container even when overflow: hidden is set on the parent. Setting min-width: 0 explicitly was the fix I needed for the character grid, and understanding why that works helped me think more carefully about how flex layout resolves sizing conflicts.

Through this coding project, i feel like i have a much better understanding of how to structure a complex webpage with multiple sections and a sticky sidebar. I also learned a lot about how to use CSS to create an engaging, editorial-style layout that supports the content. I feel like this project was a big step up in terms of complexity and polish compared to my previous projects, and I'm proud of how it turned out.

Future Improvements

If I had more time, I would add smooth scroll behavior and animated section transitions as the user navigates through the guide.

I would also expand the content for each section significantly. The Story Overview and Historical Context sections currently rely on paragraph text alone. I'd like to add pull quotes, more inline illustrations, and annotated image callouts, similar to how a real field guide or encyclopedia entry is structured.

On the technical side, I'd revisit the character cards and add a hover state that reveals a short bio for each character, i would work a little more with the spacing and also take the time to further my understanding on the coding aspect of active states, along with flexbox and the different position properties.
